## 项目概览

### 项目名称

慢性阻塞性肺疾病分级与评估系统（COPD System）

### 项目简介

本项目面向慢阻肺（COPD）患者与管理人员，提供患者基础信息管理、多量表评估（CAT、mMRC、Dyspnoea-12）、综合分级（GOLD A/B/E）、风险分层、统计报表等功能，支持患者自助录入与管理员后台管理，前后端分离部署。

## 核心业务与关键技术

### 1. 角色与权限控制（JWT + 自定义注解）

后端使用 JWT 保存登录态，在请求头 `Authorization: Bearer <token>` 中携带。通过自定义注解 `@RequireRole` 和拦截器解析 token，控制接口访问权限，例如患者端与管理员端的分离。

示例：患者与管理员混合权限控制（`PatientInfoController` 部分）：

```java
@RestController
@RequestMapping("/patient")
public class PatientInfoController {

    @Autowired
    private PatientInfoService patientInfoService;

    @Autowired
    private JwtUtil jwtUtil;

    /**
     * 患者新增自己的信息
     */
    @PostMapping("/add")
    @RequireRole("PATIENT")
    public Result<String> addPatientInfo(@RequestBody PatientInfoDTO patientInfoDTO,
                                         @RequestHeader("Authorization") String token) {
        Long userId = jwtUtil.getUserIdFromToken(token.replace("Bearer ", ""));

        PatientInfo existing = patientInfoService.getMyPatientInfo(userId);
        if (existing != null) {
            return Result.error("您已填写过患者信息，不能重复添加");
        }

        PatientInfo patientInfo = new PatientInfo();
        BeanUtils.copyProperties(patientInfoDTO, patientInfo);

        boolean success = patientInfoService.addPatientInfo(patientInfo, userId);
        return success ? Result.success("添加成功") : Result.error("添加失败");
    }
}
```

### 2. 患者信息与 GOLD 分组管理

`PatientInfoService` 基于 MyBatis-Plus 提供患者信息增删改查与分页查询，并预留 GOLD 分组和统计接口，用于随访与风险分层。

```java
@Service
public class PatientInfoService extends ServiceImpl<PatientInfoMapper, PatientInfo> {

    public boolean addPatientInfo(PatientInfo patientInfo, Long userId) {
        patientInfo.setUserId(userId);
        patientInfo.setGoldGroup("A");
        return this.save(patientInfo);
    }

    public PatientInfo getMyPatientInfo(Long userId) {
        LambdaQueryWrapper<PatientInfo> wrapper = new LambdaQueryWrapper<>();
        wrapper.eq(PatientInfo::getUserId, userId);
        return this.getOne(wrapper, false);
    }

    public IPage<PatientInfo> getPatientList(Integer pageNum, Integer pageSize, String name, String goldGroup) {
        Page<PatientInfo> page = new Page<>(pageNum, pageSize);
        LambdaQueryWrapper<PatientInfo> wrapper = new LambdaQueryWrapper<>();
        if (StringUtils.hasText(name)) {
            wrapper.like(PatientInfo::getName, name);
        }
        if (StringUtils.hasText(goldGroup)) {
            wrapper.eq(PatientInfo::getGoldGroup, goldGroup);
        }
        wrapper.orderByDesc(PatientInfo::getCreateTime);
        return this.page(page, wrapper);
    }
}
```

### 3. 前端路由与评估入口守卫

前端使用 Vue Router 定义患者端与管理员端两套布局，通过 `meta.requiresAuth` 与 `meta.role` 控制访问权限，并在进入评估问卷前检查是否已填写患者基础信息。

```js
const routes = [
  {
    path: '/patient',
    name: 'PatientLayout',
    component: () => import('@/views/patient/PatientLayout.vue'),
    meta: { requiresAuth: true, role: 'PATIENT' },
    children: [
      { path: 'home', name: 'PatientHome', component: () => import('@/views/patient/Home.vue') },
      {
        path: 'assessment/cat',
        name: 'CatAssessment',
        component: () => import('@/views/patient/CatAssessment.vue'),
        meta: { requiresPatientInfo: true }
      }
    ]
  }
]

router.beforeEach(async (to, from, next) => {
  const token = localStorage.getItem('token')
  const userRole = localStorage.getItem('userRole')

  if (to.meta.requiresAuth) {
    if (!token) {
      next('/login')
      return
    }
    if (to.meta.role && to.meta.role !== userRole) {
      next(userRole === 'ADMIN' ? '/admin' : '/patient')
      return
    }

    if (userRole === 'PATIENT' && to.meta.requiresPatientInfo) {
      const patientStore = usePatientStore(pinia)
      await patientStore.getMyPatientInfo().catch(() => {})
      if (!patientStore.hasPatientInfo) {
        next({ path: '/patient/patient-info', query: { redirect: to.fullPath } })
        return
      }
    }
  }

  next()
})
```

### 4. 患者信息表单与无障碍支持

患者信息页面使用 Element Plus 组件和 `element-china-area-data` 提供省市区级联选择，同时通过 CSS 变量控制字号，配合患者个性化字体大小设置，提升老年 COPD 患者可读性。

```vue
<template>
  <div class="patient-info-page" :style="fontSizeStyle">
    <el-card class="info-card">
      <template #header>
        <div class="card-header">
          <h2>患者信息登记</h2>
          <p class="header-subtitle">请填写您的基本信息，以便进行病情评估</p>
        </div>
      </template>

      <el-form ref="formRef" :model="form" :rules="rules" label-width="140px" class="patient-form">
        <!-- 基本信息与临床指标字段省略 -->
        <el-form-item>
          <el-button type="primary" size="large" :loading="saving" @click="handleSubmit">
            保存信息
          </el-button>
        </el-form-item>
      </el-form>
    </el-card>
  </div>
</template>

<script setup>
import { reactive, ref, computed, onMounted } from 'vue'
import { useRouter } from 'vue-router'
import { ElMessage } from 'element-plus'
import { usePatientStore } from '@/store'
import { regionData, codeToText } from 'element-china-area-data'

const router = useRouter()
const patientStore = usePatientStore()

const fontSizeStyle = computed(() => ({
  '--patient-font-size': patientStore.fontSize + 'px'
}))

const form = reactive({
  name: '',
  age: null,
  gender: '',
  birthDate: '',
  phone: '',
  province: '',
  city: '',
  district: '',
  smokingStatus: ''
})

async function handleSubmit() {
  const success = patientStore.hasPatientInfo
    ? await patientStore.updatePatientInfo(form)
    : await patientStore.savePatientInfo(form)
  if (success) {
    ElMessage.success('保存成功')
    router.push('/patient/home')
  }
}
</script>
```

## 系统部署与使用说明

> 本章节为整体技术视角的摘要说明，详细的 Windows 部署步骤请参考同目录下的 `部署方法.md`。

### 一、环境准备

- **操作系统**：Windows 10/11（64 位）
- **后端运行环境**：
  - JDK 17
  - Maven 3.9.x（如未使用 `mvnw`）
  - MySQL 8.x（已创建数据库并执行 `init.sql`）
  - Redis 6.x（可选）
- **前端运行环境**：
  - Node.js 18.x+
  - npm（随 Node.js 安装）

### 二、数据库初始化

1. 启动 MySQL 服务。
2. 在项目根目录执行初始化脚本：

```powershell
cd e:\2026work\3597-2
mysql -u root -p < .\init.sql
```

脚本会自动创建数据库 `copd_system` 及所需业务表、日志表与统计缓存表。

### 三、后端服务部署（开发/测试环境）

#### 3.1 配置数据源

在 `copd-backend/src/main/resources/application.yml` 中配置：

- 数据库连接 URL（指向 `copd_system`）
- 用户名/密码
- Redis（如使用）

#### 3.2 构建与运行

```powershell
cd copd-backend

# 构建
mvn clean package -DskipTests

# 开发/测试运行（简单方式）
mvn spring-boot:run

# 或推荐的 jar 运行方式
java -jar .\target\copd-system.war
```

默认后端监听 `8080` 端口。

### 四、前端应用部署（开发/测试环境）

#### 4.1 安装依赖

```powershell
cd copd-frontend
npm install
```

#### 4.2 配置后端 API 地址

- 推荐通过 Vite 环境变量配置（例如在 `.env.development` 中）：

```bash
VITE_API_BASE_URL=http://localhost:8080
```

前端请求封装从 `import.meta.env.VITE_API_BASE_URL` 读取后端地址。

#### 4.3 启动开发服务器

```powershell
npm run dev
```

默认前端访问地址为 `http://localhost:5173`。

### 五、生产部署示例（Nginx + 后端服务）

#### 5.1 后端

- 使用 `mvn package` 打包后，以 `java -jar` 方式运行于后台（可配合 Windows 服务/计划任务）。
- 建议：
  - 使用单独账号运行
  - 将日志重定向到文件（例如 `backend.log`）

#### 5.2 前端 + Nginx

1. 在前端项目中构建生产资源：

```powershell
cd copd-frontend
npm run build
```

2. 配置 Nginx（示例）：

```nginx
server {
    listen 80;
    server_name localhost;

    # 前端静态文件
    location / {
        root C:\copd-frontend\dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

3. 启动/重载 Nginx：

```powershell
cd C:\nginx
start nginx
.\nginx.exe -s reload
```

### 六、系统访问与验证

- 前端入口：
  - 浏览器访问 `http://localhost`（经 Nginx）
  - 或开发环境访问 `http://localhost:5173`
- 后端接口：
  - API 基础地址：`http://localhost:8080/api`（视后端实际 `@RequestMapping` 前缀而定）
- 基本验证：
  - 完成前端构建与后端启动后，可先访问登录页，完成管理员/患者登录
  - 使用管理员账号进入 `/admin/dashboard`，验证患者列表、统计等页面
  - 使用患者账号进入 `/patient/home`，依次填写患者信息并完成量表评估
