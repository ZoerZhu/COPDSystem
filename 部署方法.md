# Windows 部署指南

本文档详细介绍如何在 Windows 环境下部署慢阻肺（COPD）管理系统。

## 系统要求

### 硬件要求
- CPU: Intel Core i5 或更高
- 内存: 8GB 或更高
- 硬盘: 至少 20GB 可用空间

### 软件要求
- 操作系统: Windows 10/11 (64位)
- Node.js: 18.x 或更高
- Maven: 3.9.x
- Java: JDK 17
- MySQL: 8.0 或更高
- Redis: 6.x 或更高（可选）

---

## 一、环境准备

### 1.1 安装 Node.js

1. 访问 [Node.js 官网](https://nodejs.org/) 下载 LTS 版本
2. 运行安装程序，勾选 "Add to PATH"
3. 验证安装：
```powershell
node -v
npm -v
```

### 1.2 安装 JDK 17

1. 访问 [Oracle JDK](https://www.oracle.com/java/technologies/downloads/#java17) 下载 JDK 17
2. 运行安装程序
3. 配置环境变量：
```powershell
# 新建系统变量
JAVA_HOME = C:\Program Files\Java\jdk-17

# 编辑 PATH，添加
%JAVA_HOME%\bin
```
4. 验证安装：
```powershell
java -version
```

### 1.3 安装 MySQL 8.0

1. 访问 [MySQL 官网](https://www.mysql.com/downloads/) 下载 MySQL Installer
2. 选择 "Developer Default" 安装
3. 设置 root 密码
4. 初始化数据库：
```powershell
# 登录 MySQL
mysql -u root -p

# 创建数据库
CREATE DATABASE copd_db DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 1.4 安装 Redis（可选）

1. 访问 [Redis 官网](https://redis.io/download) 下载 Windows 版本
2. 或使用 Memurai/Redis Windows 端口
3. 启动 Redis 服务：
```powershell
redis-server
```

## 二、后端部署

### 2.1 配置数据库连接

编辑 `copd-backend/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/copd_db?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: root
    password: 你的MySQL密码

# Redis 配置（可选）
  redis:
    host: localhost
    port: 6379
```

### 2.2 初始化数据库

执行数据库初始化脚本（以项目根目录为起点示例）：

```powershell
# 进入项目根目录（按实际路径调整）
cd e:\2026work\3597-2

# 执行初始化脚本（按项目实际存放位置调整）
mysql -u root -p copd_db < .\copd-backend\init.sql
```

### 2.3 启动后端服务

```powershell
cd copd-backend

# 使用 Maven 构建
mvn clean package -DskipTests

# 运行
mvn spring-boot:run
```

或使用 IDE（如 IntelliJ IDEA）直接运行 `CopdBackendApplication.java`。

后端默认端口：**8080**

生产环境建议（可选）：

```powershell
# 若有 application-prod.yml，可启用生产 profile（按项目实际为准）
setx SPRING_PROFILES_ACTIVE prod
```

更推荐的“生产运行方式”（避免依赖 Maven 常驻）：

```powershell
cd copd-backend

# 先打包得到 jar（jar 名称按实际生成的为准）
mvn clean package -DskipTests

# 运行 jar（可配合 SPRING_PROFILES_ACTIVE / application-prod.yml）
java -jar .\target\*.jar
```

## 三、前端部署

### 3.1 安装依赖

```powershell
cd copd-frontend
npm install
```

### 3.2 配置 API 地址

优先使用环境变量配置（Vite 项目常用方式；如项目不是 Vite，请按实际代码为准）。

在 `copd-frontend` 下创建/修改 `.env.development`：

```bash
VITE_API_BASE_URL=http://localhost:8080
```

如项目当前是写死在代码里，则编辑 `copd-frontend/src/utils/request.js`：

```javascript
const baseURL = 'http://localhost:8080'  // 后端API地址
```

### 3.3 开发模式运行

```powershell
npm run dev
```

前端默认端口：**5173**

### 3.4 生产环境构建

```powershell
npm run build
```

构建产物在 `dist` 目录，可部署到 Nginx 或任何静态服务器。

## 四、Nginx 部署（可选）

### 4.1 安装 Nginx

1. 访问 [Nginx 官网](https://nginx.org/en/download.html) 下载 Windows 版本
2. 解压到 `C:\nginx`

### 4.2 配置 Nginx

编辑 `C:\nginx\conf\nginx.conf`：

```nginx
server {
    listen 80;
    server_name localhost;

    # 前端静态文件
    location / {
        root C:\copd-frontend\dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api/ {
        # 保留 /api 前缀转发（与前端请求路径保持一致）
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 4.3 启动 Nginx

```powershell
cd C:\nginx
start nginx
```

### 4.4 停止/重载 Nginx

```powershell
cd C:\nginx

# 优雅停止
.\nginx.exe -s quit

# 快速停止
.\nginx.exe -s stop

# 重载配置
.\nginx.exe -s reload
```

## 六、常见问题

### 6.1 端口被占用

```powershell
# 查看端口占用
netstat -ano | findstr 8080

# 结束占用进程
taskkill /PID <进程ID> /F
```

### 6.2 数据库连接失败

1. 确认 MySQL 服务已启动
2. 检查用户名密码
3. 确认数据库已创建

### 6.3 前端无法访问后端

1. 检查后端是否正常运行
2. 检查 API 地址配置
3. 检查防火墙设置

### 6.4 如何查看日志定位问题

- **后端**：若用 `java -jar` 启动，控制台会输出启动与报错信息；也可在启动命令中将输出重定向到文件：

```powershell
java -jar .\target\*.jar *> backend.log
```

- **Nginx**：查看 `C:\nginx\logs\access.log` 与 `C:\nginx\logs\error.log`

## 七、安全建议

1. **修改默认密码**：首次部署后修改 MySQL root 密码
2. **配置防火墙**：只开放必要端口（80, 443, 8080）
3. **HTTPS 部署**：生产环境建议配置 HTTPS
4. **定期备份**：定期备份数据库

备份与恢复示例：

```powershell
# 备份
mysqldump -u root -p copd_db > copd_db_backup.sql

# 恢复
mysql -u root -p copd_db < copd_db_backup.sql
```
